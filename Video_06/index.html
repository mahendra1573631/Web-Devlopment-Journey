Block-level Elements

  • Start on a new line: Block elements always begin on a new line, effectively creating a "block" of content.
  • Take full available width: By default, they expand horizontally to occupy the entire available width of their parent container.
  • Can have height, width, padding, and margin: You can explicitly set dimensions and spacing around block elements.
  • Can contain other block and inline elements: They serve as containers for structuring content.
  • Examples: p tag (paragraph), div tag (division), h1-h6 tag (headings), ul tag, ol tag, li tag (lists), form tag, header tag, footer tag, section tag.

Inline Element

  • Do not start on a new line: Inline elements flow within the existing line of content.
  • Take only necessary width: They only occupy the horizontal space required by their content.
  • Cannot typically have explicitly set height and width: While some inline elements like img tag are "replaced elements" and can have dimensions, most inline elements do not allow explicit height and width settings.
  • Examples: span tag (generic inline container), a tag (anchor/link), strong tag (strong importance), em tag (emphasis), img tag (image), br tag (line break), input tag.

Key difference between inline and block line elements

Feature Block Elements Inline Elements
New Line Always start on a new line Do not start on a new line
Width Take full available width Take only necessary width
Dimensions Can set height,width,padding,margin Cannot typically set height,width(except replaced elements)
Containment Can contain block and inline elements Cannot contain block-level elements
